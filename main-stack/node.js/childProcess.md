### Child process

Child process (дочерний процесс) - механизм для создания и управления дополнительными процессами вне основного потока выполнения. Это позволяет выполнять системные команды, запускать другие программы или скрипты параллельно. 

Основные [причина использования](./childprocess/reason.js) дочернего процесса:
1. Выполнения системных команд - 
2. Запуск внешних программ/библиотек
3. Параллельная обработка CPU-интенсивных задач. 
4. Изоляция и безопасность
- Критические ошибки в дочернем процессе не крашат основное приложение
- Возможность sandbox-режима

Существует несколько способов создания child process
1. [exec()](./childprocess/create/exec.js) - для выполнения команд с буферизацией вывода. К характеристикам относится буферизирует весь вывод, возвращает результат после завершения команды, максимум 200кб вывода по умолчанию. 
2. [spawn()](./childprocess/create/spawn.js) - потоковое выполнение команд
3. [fork()](./childprocess/create/fork.js) - специализированный spawn для Node.js скриптов
4. [execFile()](./childprocess/create/execFile.js) - выполнение файлов без shell

| Метод | Shell | Потоковый вывод | IPC | Описание (когда использовать) | 
| ---- | ---- | ---- | ---- | ---- |
| exec() | + | - | - | Когда нужно выполнить короткую системную команду и получить весь результат сразу. |
| spawn() | - (по умолчанию) | - | + | Когда нужно обрабатывать большой объем данных или длительные процессы.  |
| fork() | - | + | + | Когда нужно запустить другой Node.js скрипт с двусторонней связью. |
| execFile() | - | - | - | 	Когда нужно запустить исполняемый файл напрямую, минуя shell. | 

Основные свойства и события через `const child = spawn('ls', ['-la'])`: 

| Свойства | Что делает | 
|-----|----------|
| child.stdin | Поток для ввода (Writable Stream) |
| child.stdout | Поток для вывода (Readable Stream) | 
| child.stderr | Поток для ошибок (Readable Stream) | 
| child.pid | Поток для ошибок (Readable Stream) |
| child.killed | Был ли процесс убит | 
| child.exitCode |  Код завершения | 
| child.on('close', (code, signal) => { ... }) | Все потоки закрыты | 
| child.on('exit', (code, signal) => { ... }) | Процесс завершен, но потоки могут быть открыты |
| child.on('error', (err) => { ... }) | Ошибка при создании/запуске процесса | 
| child.on('message', (message) => { ... }) | Только для fork() - сообщение через IPC | 
| child.on('disconnect', () => { ... }) | IPC канал закрыт |
