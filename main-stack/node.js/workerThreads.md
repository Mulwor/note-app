### Worker threads 

Worker Threads - это модуль Node.js, который позволяет выполнять JavaScript код в параллельных потоках внутри одного процесса. Код в рабочем потоке выполняется изолированно, что предотвращает блокировку основного потока Event Loop.

Worker Threads не подходят для I/O-задач (сеть, файловая система). Для них Event Loop основного потока эффективнее. Они созданы именно для тяжелых CPU-операций (криптография, обработка изображений/видео, сложные математические расчеты).

Этот модуль также позволяет запускать несколько экземпляров одного приложения Node.js, которые могут распределять вычислительные нагрузки между ядрами процессора.

У worker.on('') есть несколько событий: 
1\. 'online' - worker запустился и выполнил начальную загрузку
2\. 'message' - получено сообщение от worker
3\. 'error' - произошла ошибка в worker
4\. 'exit' - worker завершил работу

Помимо события `.on` у `worker` threads есть также свои методы и свойства. 
| Метод | Что делает данный метод | 
| ----- | ----------------------- |
| `.postMessage(value)` | Отправляет данные |
| `.terminate()`   | Асинхронно завершает работу воркера (возвращает Promise). | 
| `.ref()` - режим по умолчанию / `.unref()` | Это методы, которые контролируют должен ли Node.js ждать завершения этого воркера перед тем, как завершить работу всей программы. Когда ты создаешь воркер, Node.js по умолчанию считает его активной задачей, которую нужно дождаться. Программа не завершится, пока жив хотя бы один воркер. |
| `threadId` | Уникальный id worker |
| `.stdin, .stdout, .stderr` |	(Потоки) Доступ к стандартным потокам ввода/вывода воркера, если они были разрешены при его создании. |
