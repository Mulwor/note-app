`useTransition` - хук, который обновляет состояние без блокировки UI (пользовательского интерфейса). Предназначен для оптимизации производительности при работе с "тяжелыми" обновлениями состояния.

Он позволяет помечать некоторые обновления состояния как «не срочные» (transition), чтобы React не блокировал интерфейс во время их выполнения. Пока идет загрузка данных или перерисовка сложного компонента, интерфейс остается отзывчивым (можно кликать, вводить текст и т.д.).

Он не принимает никаких параметров, а синтаксис его таков - `const [isPending, startTransition] = useTransition();`, где

- `isPending` — булево значение, true пока transition ещё выполняется. Используется чтобы показать индикатор загрузки.
- `startTransition(fn)` — функция, которая принимает колбэк с обновлением состояния и помечает его как «не срочное».

### Как работает под капотом

React использует приоритеты обновлений. Обычные `setState` — высокий приоритет (срочные). Обновления внутри `startTransition` — низкий приоритет. Если появляется срочное обновление (например, пользователь печатает), React прерывает transition и сначала обрабатывает срочное.
